# Construction Tracker - Module Guide

This guide explains the purpose and usage of each module in the system.

## Employee Profiles
**Purpose:** Manage worker documentation and personal information with an approval workflow.

**Who Uses It:**
- Workers: Submit their profile documents (ID, certifications, etc.)
- Admins: Review and approve/reject submitted profiles

**Key Features:**
- Document upload and management
- Status tracking (Draft → Submitted → Approved/Rejected)
- Soft delete with restore capability
- Review notes and feedback

**When to Use:**
- Onboarding new workers
- Annual document renewals
- Compliance audits

---

## Work Progress
**Purpose:** Visual dashboard showing overall work order completion statistics.

**Who Uses It:**
- Managers and supervisors to track job progress
- Admins for high-level oversight

**Key Features:**
- Completion rate visualization
- Status breakdown (Completed, In Progress, Pending, On Hold)
- Recent work orders list
- Quick statistics cards

**When to Use:**
- Daily standups to check progress
- Weekly reviews
- Identifying bottlenecks

---

## Time Management / Time Logs
**Purpose:** Track employee clock-in/clock-out times with signatures.

**Who Uses It:**
- Workers: Clock in/out via mobile
- Admins/HR: Review and manage time entries
- Payroll: Export data for payroll processing

**Key Features:**
- Digital signatures for verification
- Duration calculation
- Edit/audit trail
- Export capabilities

**When to Use:**
- Daily time tracking
- Payroll processing
- Dispute resolution
- Compliance reporting

---

## All Time Logs (Admin View)
**Purpose:** Administrative interface for viewing and editing all employee time entries.

**Who Uses It:**
- Admins and HR staff only

**Key Features:**
- View all employee time logs
- Edit timestamps and notes
- Delete incorrect entries
- Filter by date and employee

**When to Use:**
- Correcting time entry mistakes
- Auditing time records
- Resolving disputes

---

## Work Logs
**Purpose:** Daily work reports submitted by workers describing completed tasks.

**Who Uses It:**
- Workers: Submit daily summaries
- Supervisors: Review and provide feedback
- Managers: Track productivity

**Key Features:**
- Task descriptions
- Photo documentation
- Status workflow (Pending → Reviewed → Approved/Needs Revision)
- Admin feedback system

**When to Use:**
- End-of-day reporting
- Progress tracking
- Quality assurance

---

## Payroll Report
**Purpose:** Generate payroll reports based on time logs and work completed.

**Who Uses It:**
- HR and payroll administrators
- Managers approving payroll

**Key Features:**
- Hours worked calculation
- Pay rate application
- Export to payroll systems
- Pay stub uploads

**When to Use:**
- Weekly/bi-weekly payroll runs
- Pay stub distribution
- Payroll audits

---

## Schedule Management
**Purpose:** Create and manage worker schedules for buildings.

**Who Uses It:**
- Schedulers and managers
- Workers: View their assigned schedules

**Key Features:**
- Calendar and list views
- Worker assignment
- Time conflict detection
- Status tracking (pending, in-progress, completed)

**When to Use:**
- Weekly schedule planning
- Assigning workers to buildings
- Tracking schedule completion

---
